Reth runs on Linux and macOS. Windows support is tracked but not officially released.

There are three ways to install Reth:

  <Card title="Pre-built binaries" icon="download" href="https://github.com/paradigmxyz/reth/releases">
    Download static executables for Linux, macOS, or Windows from GitHub Releases. Fastest way to get started.

    Pull the official image from GitHub Container Registry and run Reth in a container with a single command.
  </Card>
  <Card title="Build from source" icon="code" href="/installation/source">

If you have Docker installed, the [Docker Compose](/installation/docker#using-docker-compose) setup gets Reth, Lighthouse (consensus client), Prometheus, and Grafana running and syncing with a single command.
</Note>

## Pre-built binaries

[Pre-compiled binaries](https://github.com/paradigmxyz/reth/releases) are available for Windows, macOS, and Linux as static executables. This is the recommended method for most users.

**macOS / Linuxbrew** users can install via Homebrew:

```bash
brew install paradigmxyz/brew/reth
```

**Arch Linux** users can install from the AUR:

```bash
paru -S reth        # Stable release
paru -S reth-git    # Latest from git
```

### Verify your installation

After installing, confirm Reth is working correctly:

```bash
reth --version
```

You should see the version number printed to the terminal.

### Signature verification

You can verify the integrity of any release using GPG. First, import the release signing key:

```bash
gpg --keyserver keyserver.ubuntu.com --recv-keys 50FB7CC55B2E8AFA59FE03B7AA5ED56A7FBF253E
```

Then verify the signature file that accompanies each release:

```bash
gpg --verify reth-v1.0.0-x86_64-unknown-linux-gnu.tar.gz.asc \
             reth-v1.0.0-x86_64-unknown-linux-gnu.tar.gz
```

Replace the filenames with those matching your downloaded release.
