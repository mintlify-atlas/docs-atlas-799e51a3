## What is Reth?

Reth is an **Ethereum Execution Layer (EL)** implementation compatible with all consensus clients that support the [Engine API](https://github.com/ethereum/execution-apis/tree/main/src/engine). It is suitable for mission-critical deployments including staking, high-uptime RPC services, MEV extraction, on-chain indexing, and custom EVM chains.

## Quick start

<Steps>
  <Step title="Install Reth">
    Download a pre-built binary for your platform:
    ```bash
    # Linux / macOS — download from GitHub Releases
    curl -L https://github.com/paradigmxyz/reth/releases/latest/download/reth-$(uname -s)-$(uname -m).tar.gz | tar xz
    ```
    Or use Docker: `docker pull ghcr.io/paradigmxyz/reth`
  </Step>
  <Step title="Start a node">
    Run Reth against Ethereum mainnet alongside any consensus client:
    ```bash
    reth node \
      --authrpc.jwtsecret /path/to/jwt.hex \
      --authrpc.addr 127.0.0.1 \
      --authrpc.port 8551
    ```
  </Step>
  <Step title="Query the RPC">
    Once synced, query your node via JSON-RPC:
    ```bash
    cast block-number --rpc-url http://localhost:8545
    ```
  </Step>
</Steps>

## Use Reth as a library

Add the Ethereum meta-crate to your `Cargo.toml` and build custom nodes with the Node Builder API:

```toml Cargo.toml
[dependencies]
reth-ethereum = { git = "https://github.com/paradigmxyz/reth" }
```

```rust main.rs
use reth_ethereum::node::{EthereumNode, NodeBuilder};

let handle = NodeBuilder::new(config)
    .with_types::<EthereumNode>()
    .with_components(EthereumNode::components())
    .launch()
    .await?;
```
