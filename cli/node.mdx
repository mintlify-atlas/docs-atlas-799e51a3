---
title: "reth node"
description: "Start the Reth Ethereum execution node. Full flag reference for networking, RPC, engine API, transaction pool, pruning, and more."
---

The `reth node` command starts the Ethereum execution client. It connects to a consensus client over the Engine API, participates in the P2P network, and serves JSON-RPC requests.

```bash
reth node --help
```

## Quick examples

<Tabs>
  <Tab title="Archive node">
    Default mode. Retains all historical state. Requires the most disk space.

    ```bash
    reth node \
      --authrpc.jwtsecret /path/to/jwt.hex \
      --authrpc.addr 127.0.0.1 \
      --authrpc.port 8551
    ```
  </Tab>
  <Tab title="Full node">
    Prunes historical state. Significantly less disk space with broad RPC support.

    ```bash
    reth node --full \
      --authrpc.jwtsecret /path/to/jwt.hex
    ```
  </Tab>
  <Tab title="With HTTP RPC">
    Enable the HTTP JSON-RPC server on the default port (8545).

    ```bash
    reth node \
      --http \
      --http.api eth,net,web3,debug \
      --authrpc.jwtsecret /path/to/jwt.hex
    ```
  </Tab>
  <Tab title="Dev / testing">
    Start a local PoA chain with pre-funded accounts. No consensus client needed.

    ```bash
    reth node --dev --http --http.api eth,net,web3
    ```
  </Tab>
</Tabs>

## General options

| Flag | Default | Description |
|---|---|---|
| `--config <FILE>` | — | Path to TOML configuration file |
| `--chain <CHAIN_OR_PATH>` | `mainnet` | Chain to run. Built-in: `mainnet`, `sepolia`, `holesky`, `hoodi`, `dev`. Or path to a custom chain spec JSON |
| `--instance <INSTANCE>` | — | Run multiple nodes on the same machine. Offsets all default ports. Max 200 instances |
| `--with-unused-ports` | — | Bind all sockets to random unused OS ports. Mutually exclusive with `--instance` |

## Datadir

| Flag | Default | Description |
|---|---|---|
| `--datadir <DATA_DIR>` | OS data dir | Root directory for all Reth data. Linux: `$HOME/.local/share/reth/`, macOS: `$HOME/Library/Application Support/reth/` |
| `--datadir.static-files <PATH>` | — | Absolute path for static file storage |
| `--datadir.rocksdb <PATH>` | — | Absolute path for RocksDB storage |
| `--datadir.pprof-dumps <PATH>` | — | Absolute path for pprof dump files |

## Metrics

| Flag | Default | Description |
|---|---|---|
| `--metrics <PROMETHEUS>` | — | Enable Prometheus metrics at the given address (e.g., `0.0.0.0:9001`) |
| `--metrics.prometheus.push.url <URL>` | — | Push gateway URL for Prometheus remote write |
| `--metrics.prometheus.push.interval <SECONDS>` | `5` | Push interval in seconds |

## Pruning

Run a full node (pruned) or configure fine-grained pruning:

| Flag | Description |
|---|---|
| `--full` | Run a full node. Stores only the most recent safe-distance block states |
| `--minimal` | Maximum pruning with smallest static files. Prunes sender recovery, tx lookup, receipts |
| `--prune.block-interval <N>` | Minimum pruning interval in blocks |
| `--prune.sender-recovery.full` | Prune all sender recovery data |
| `--prune.sender-recovery.distance <N>` | Keep sender recovery only for the last N+1 blocks |
| `--prune.transaction-lookup.full` | Prune all transaction lookup data |
| `--prune.receipts.full` | Prune all receipt data |
| `--prune.receipts.pre-merge` | Prune receipts before the Merge block |
| `--prune.account-history.full` | Prune all account history |
| `--prune.storage-history.full` | Prune all storage history data |

## Engine API setup for consensus clients

The Engine API is the interface between the execution layer (Reth) and the consensus layer (Lighthouse, Prysm, Teku, Nimbus, etc.).

<Steps>
  <Step title="Generate a JWT secret">
    If you don't already have a JWT secret, Reth generates one automatically. To specify your own:

    ```bash
    openssl rand -hex 32 | tee /path/to/jwt.hex
    ```
  </Step>
  <Step title="Start Reth with the Engine API">
    ```bash
    reth node \
      --authrpc.jwtsecret /path/to/jwt.hex \
      --authrpc.addr 127.0.0.1 \
      --authrpc.port 8551
    ```
  </Step>
  <Step title="Configure your consensus client">
    Point your consensus client at Reth's Engine API endpoint. Example for Lighthouse:

    ```bash
    lighthouse bn \
      --execution-endpoint http://localhost:8551 \
      --execution-jwt /path/to/jwt.hex
    ```
  </Step>
</Steps>

<Warning>
  The JWT secret must be identical in both Reth and the consensus client. A mismatch will cause authentication failures and prevent the node from syncing.
</Warning>
