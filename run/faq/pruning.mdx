By default, Reth runs as an **archive node**, which stores all historical blocks and state from genesis. Reth also supports running as a **full node** or a **pruned node** to reduce disk usage.

<Note>
  The node type you choose when you first run Reth **cannot be changed** after the initial sync. Converting between archive, full, and pruned modes requires a full resync.
</Note>

## Basic concepts

| Mode             | Description                                                                                                          |
| ---------------- | -------------------------------------------------------------------------------------------------------------------- |
| **Archive node** | Stores all historical data from genesis. Every block and every historical state is available for querying.           |
| **Full node**    | Stores the latest state and historical data for only the last 10,064 blocks. Uses `--full` flag.                     |
| **Pruned node**  | Historical data pruned partially or fully via custom configuration in `reth.toml`.                                   |
| **Minimal node** | A preconfigured pruned profile that minimizes disk usage. Uses `--minimal` flag.                                     |

## Modes

### Archive node

Archive mode is the default. Follow the [running on mainnet or official testnets](/run/ethereum) guide — no extra flags required.

### Full node

Add the `--full` flag when starting Reth:

```bash
reth node \
    --full \
    --authrpc.jwtsecret /path/to/secret \
    --authrpc.addr 127.0.0.1 \
    --authrpc.port 8551
```

A full node prunes sender recovery, most receipts (preserving Beacon Deposit Contract receipts), and account and storage history older than 10,064 blocks. Transaction lookup is not pruned.

### Pruned node

To use a custom pruning configuration, edit `reth.toml` and add a `[prune]` section. Then run Reth the same way as an archive node. See the [configuration reference](/run/configuration#the-prune-section) for all available options.

### Minimal storage mode

Add the `--minimal` flag to minimize disk usage:

```bash
reth node \
    --minimal \
    --authrpc.jwtsecret /path/to/secret \
    --authrpc.addr 127.0.0.1 \
    --authrpc.port 8551
```

Minimal mode fully prunes sender recovery, transaction lookup, and receipts, and retains only the last 10,064 blocks of account history, storage history, and block bodies.

## Disk space

All numbers are approximate, as of April 2024 at block ~19.6M on mainnet.

### Archive node

An archive node requires at least **2.14 TB**.

### Full node

A full node requires at least **1.13 TB**.

### Pruned node

The table below shows how much space each segment frees when pruned completely:

| Segment            | Space freed |
| ------------------ | ----------- |
| Sender Recovery    | 85 GB       |
| Transaction Lookup | 200 GB      |
| Receipts           | 250 GB      |
| Account History    | 235 GB      |
| Storage History    | 590 GB      |

Combine segments to calculate total savings for your pruning configuration.

## Full node TOML equivalent

The `--full` flag is equivalent to the following `reth.toml` configuration:

```toml
[prune]
block_interval = 5

[prune.segments]
sender_recovery = "full"
# transaction_lookup is not pruned
receipts = { before = 11052984 } # Beacon Deposit Contract deployment block

account_history = { distance = 10_064 }
storage_history = { distance = 10_064 }

[prune.segments.receipts_log_filter]
# Preserve receipts with logs from the Beacon Deposit Contract
"0x00000000219ab540356cbb839cbe05303d7705fa" = { before = 11052984 }
```
