Reth is an Ethereum execution client. Once installed, you can run it on Ethereum mainnet, public testnets, or a private local network. This section covers everything you need to operate a node — from initial setup through monitoring in production.

<Note>
  Reth is an **execution client** only. After Ethereum's transition to Proof of Stake, you must also run a **consensus client** (such as Lighthouse, Teku, or Prysm) alongside Reth to sync any post-Merge network. The two clients communicate over the Engine API using a shared JWT secret.
</Note>

## Node modes

Reth supports two primary operating modes:

| Mode | Flag | Description |
|---|---|---|
| Archive node | `reth node` | Retains all historical state. Largest storage footprint, full historical query support. |
| Full node | `reth node --full` | Prunes historical state. Smaller footprint, suitable for most use cases. |

For a detailed comparison of storage requirements and pruning options, see the [Pruning & Full Node](/run/faq/pruning) FAQ.

## Prerequisites

Before starting Reth, ensure you have:

- Reth installed — see the [installation guide](/installation/overview)
- A supported consensus client installed (Lighthouse, Teku, Prysm, Nimbus, or Lodestar)
- Sufficient disk space — see [System Requirements](/run/system-requirements)
